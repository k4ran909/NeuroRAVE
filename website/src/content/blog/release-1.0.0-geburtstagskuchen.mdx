These are the release notes for Neuro RAVE 1.0.0 aka "Geburtstagskuchen"

This release marks the 2 year anniversary of the project, the first commit was on the 22nd January 2022 by Alex McLean.

If you generally need a heads up on what happened to Neuro RAVE in the last year, read the [2023 recap](/blog/#year-2)

import { Youtube } from '@src/components/Youtube';

A lot has happened since then, and also since the last release 16 weeks ago.

Let me write up some of the highlights:

## Breaking Change

This version changes the default cps value from 1 to 0.5 to give patterns a little bit more time by default.
If you find your existing patterns to be suddenly half the speed, just add a `setcps(1)` to the top and it should sound as it did before!

- make 0.5hz cps the default by @yaxu in PR #931

## New Domain

Neuro RAVE is now available online. The old domain still works but you might not get the most recent version.

- replace strudel.tidalcycles.org with the new domain by @felixroos in PR #768

## Neuro RAVE on Mastodon

Neuro RAVE now has a mastodon presence: https://social.toplap.org/@strudel

## Slider Controls

The new `slider` function inlines a draggable slider element into the code, bridging the gap between code and GUI.

- widgets by @felixroos in PR #714
- Slider afterthoughts by @felixroos in PR #723
- add xfade by @felixroos in PR #780

## Improved MIDI integration

Pattern params [can now be controlled with cc messages](https://www.youtube.com/watch?v=e2-Sv_jjDQk) + you can now send a MIDI clock to sync your DAW with strudel.

- Midi in by @felixroos in PR #699
- add midi clock support by @felixroos in PR #710

## hydra

[hydra](https://hydra.ojack.xyz), the live coding video synth can now be used directly inside the Neuro RAVE REPL.

- Hydra integration by @felixroos in PR #759
- add options param to initHydra by @kasparsj in PR #808
- Hydra fixes and improvements by @atfornes in PR #818

## Vanilla REPL

The codemirror editor and the repl abstraction have been refactored from react to vanilla JS!
This should give some performance improvements and less dependency / maintenance burden:

- Vanilla repl 2 by @felixroos in PR #863
- Vanilla repl 3 by @felixroos in PR #865
- more work on vanilla repl: repl web component + package + MicroRepl by @felixroos in PR #866
- main repl vanillification by @felixroos in PR #873
- final vanillification by @felixroos in PR #876
